# 1. Описание классов

Реализована иерархия классов `Apartment`, `House`, `Penthouse` недвижимости на основе базового класса `Property` и коллекции `Agency`

---

## Назначение

Создана система классов, которая:

- описывает разные типы недвижимости  
- переиспользует общий код базового класса  
- реализует разное поведение через полиморфизм  
- интегрируется с коллекцией из ЛР-2  

---

## Базовый класс

### `Property`

Описывает общие характеристики недвижимости:

- цена  
- площадь  
- адрес  
- аренда / продажа  
- срок аренды  
- активность  

---

## Дочерние классы

### `Apartment`

Дополнительные атрибуты:

- этаж  
- балкон  
- количество комнат  

---

### `House`

Дополнительные атрибуты:

- количество этажей  
- наличие лифта  
- площадь участка  
- количество комнат  

---

### `PentHouse`

Дополнительные атрибуты:

- количество этажей  
- балкон  
- количество комнат  

---

## Наследование

- все классы наследуются от `Property`  
- используется `super()`  
- код базового класса не дублируется  

---

## Переопределение методов

- `__str__` — расширенный вывод для каждого типа недвижимости  

---

## Полиморфизм

Метод:

- `calculate()`

Работает по-разному для разных классов:

- `Apartment` → налог 5%  
- `House` → налог 10%  
- `PentHouse` → налог 15%  

Вызов одинаковый:

```python
for item in agency:
    print(item.calculate())
```

## Методфы определения типов объектов
- `get_only_apartments()`
- `get_only_houses()`
- `get_only_penthouses()`

### Работа с типами

Используется:
isinstance() — для определения типа объекта
